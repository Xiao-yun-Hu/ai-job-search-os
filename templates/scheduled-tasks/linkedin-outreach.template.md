# Task: LinkedIn Cold Outreach (Semi-automated)

> NOT a cron task — this is a manual / on-demand workflow because each cold DM should be reviewed before sending.
> Purpose: For Tier A targets (overseas), send LinkedIn DMs to founder / CTO / hiring manager.

## When to run
- After daily Tier A list generated by morning task
- When you want to send a batch (e.g., 5-7 DMs in one sitting)
- After vibe research is complete on each target

## Pre-flight
- Read `[/path/to]/research/vibe_*.md` (vibe research per company)
- Read candidate's narrative pillars from `memory/project_candidate_profile_*.md`
- Have LinkedIn open and logged in (via Chrome MCP or similar)

## Phase 1: Compile DM list

For each Tier A company that hasn't been DM'd yet:
- Identify recommended LinkedIn target (founder / CTO / IC peer per vibe research)
- Pull their LinkedIn profile URL
- Note 1-2 hooks from their public material (recent blog post, podcast appearance, X thread)

## Phase 2: Compose 3 message variants per target

Per target, generate 3 versions:

**Version 1: Systems-design hook**
```
Hi [Name], saw your post on [specific topic]. I'm building [candidate's flagship multi-agent / decision system project] — would love to compare notes on [specific architectural angle they've discussed]. Open to a 15-min chat?
```

**Version 2: Eval / observability hook**
```
Hi [Name], read your blog on [evaluation methodology / observability]. I'm wrestling with [specific candidate problem] in production at [scale] — your framing of [their specific point] really clicked. Coffee chat possible?
```

**Version 3: Business / outcome hook**
```
Hi [Name], your team's [specific customer outcome / case study] is exactly the kind of work I want to do next. I've spent the last [X] years compressing [research/operation] cycles by [Y]× at [scale] — would value 15 min to learn how you scope [specific problem]?
```

> Each ≤120 chars body if possible. Add custom closing.
> User reviews + picks one (or modifies) before sending.

## Phase 3: User Review (manual)

For each message:
- Read it as if you were the recipient
- Does it have a real hook? (NOT just "love your work")
- Does it state your specific value? (NOT generic "AI experience")
- Does it have a clear ask? (15 min, specific)

If any "no" → revise.

## Phase 4: Send (in browser)

For each approved message:
1. Navigate to LinkedIn profile
2. Click "Message" or "Connect with note"
3. Paste message, send

LinkedIn limits:
- ~100 connection requests/week (cold)
- Free tier limits InMail
- Premium has more

## Phase 5: Log (manual)

Append to `applications.jsonl`:

```json
{
  "ts": "...",
  "company": "...",
  "title": "[role being explored]",
  "channel": "linkedin_cold_dm",
  "outreach_target_name": "...",
  "outreach_target_title": "...",
  "message_version": "v1_systems | v2_eval | v3_business",
  "tier": "A",
  "outcome": "sent",
  "linkedin_url": "..."
}
```

This goes into the same funnel-tracking pipeline as job-board applications.

## Tips

- **Don't blast the same message** to all 5 targets — each should reference their specific public work
- **Wait 5-7 days** before following up if no reply (follow-up: shorter, friendlier, "wanted to bump this up")
- **Don't add to LinkedIn first then DM** — be transparent: cold outreach with hook, ask for connection in the message
- **Track which version (v1/v2/v3) had higher reply rate** — but don't act on it until ≥10 sends per version

## Hard prohibitions
- ❌ Don't auto-send (every DM is reviewed)
- ❌ Don't paste candidate's resume into the DM (too much, too soon — save for the 15-min chat)
- ❌ Don't mention salary expectations in cold DM
- ❌ Don't connect-then-no-message (looks transactional)
